# Security Rules

- 소스코드에 비밀키, API 키, 비밀번호, 토큰을 절대 하드코딩 금지
- 모든 시크릿은 환경변수로 로드
- 로컬 개발은 `.env` 사용 (.env 파일은 절대 커밋 금지)
- API 경계에서 모든 외부 입력을 검증 (Pydantic, Zod 등)
- SQL: ORM 또는 파라미터화된 쿼리만 사용 — 문자열 직접 조합 금지
- API 엔드포인트: 처리 전 반드시 인증 검증
- 파일 업로드: 처리 전 파일 타입과 크기 검증
- 로그 위생: 시크릿, 토큰, 민감한 데이터는 절대 로깅 금지
- 비밀번호: 최소 8자, bcrypt/argon2 해싱
- CORS: 허용 오리진 명시적 제한
